00b · Design Rationale The Case for Newsprint

Why a newspaper, not a dashboard.

Max Mahon is a weekend research tool, not a trading cockpit. The user will open it once a week to read sixty-odd dividend candidates and decide whether to rebalance. That reading posture — slow, literate, weekend-coffee — belongs to the weekend edition of a financial paper, not to a real-time admin panel. Every layout choice here proceeds from that brief.

Cards behave like article stubs. The full report behaves like a Saturday long-read, with byline, drop cap, pull quote, and a two-column set text. The score is treated as a lead number in the way Barron's treats a fund rating. Severity — hold, review, consider-exit — is expressed through rule weight and italic, not through traffic-light color. This keeps the tool readable, weighted, and — by our own house rule — mobile-forgiving.

One accent, no decoration.

Color discipline was the hardest single decision. The existing v5 mockup used forest green, burgundy, amber, navy, and gold. That palette is a dashboard palette. For the weekly edition we pulled all of them and kept a single oxblood as the accent — used only for the current navigation indicator, the new-entrant ribbon, the exit-severity border, and a single highlighted data point on each chart. Everything else is ink on cream paper, in three shades of gray. If the accent ever needs to compete with another color, the design has failed.

Numbers live in monospace.

All numerical values render in JetBrains Mono with tabular numerals. Everything else is Lora for running text and Playfair Display for headlines and scores. Thai support falls to IBM Plex Serif Thai when a Latin serif cannot render a glyph. Score 74 is the same weight and density as score 9, ten million, or 324B — because in a weekly screen the digits are the story.

Rule lines instead of boxes.

There are no card shadows. No box radii above two pixels. No gradients. Hierarchy is enforced by horizontal rules (thin, thick, double), vertical column rules, and the occasional three-em ornament. This matches how pre-1980s financial weeklies organized hundreds of rows of tabular data — and it survives a print-to-PDF export, which the dashboard style does not.

Mobile preserves identity, not layout.

The mobile edition is not the desktop scaled. Masthead compresses to a single stacked block. Top-nav becomes a five-icon bottom bar with numeric chapter labels. Tabs in the simulator become a dropdown. Cards collapse to one column. The drop cap shrinks to 3.6 em. But the fonts, the rule discipline, and the oxblood accent survive intact — a reader should recognize the paper on either surface.

What was deliberately not built.

No dark mode (the brief is a newspaper; newspapers are not dark). No animation beyond subtle hover. No data export, print stylesheets, or accessibility-specific variants yet — this is a direction mockup, not a release. Charts use sample data that approximates realistic Thai dividend patterns; the numbers are illustrative.
